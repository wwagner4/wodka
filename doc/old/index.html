goal

Wodka is the implementation of a genetic algorithm for breeding soda robots. It is designed as platform that allows configuration of the genetic algorithm in an easy and flexible way.

The idea is that wodka provides the infrastructure that is needed to breed soda robots, to allow you focusing on your ideas without having to think about technical details.

features - current

  • Interface for implementing different genetic representations (genotypes) of soda robots.
  • Interface for implementing different selection strategies.
  • Java class representation of soda models.
  • Marshaller for converting models to xml and xml to models.
  • Racer classes for easily testing models on the soda race application.
  • Persistence layer for saving and reloading the genetic algorithm at any state of evaluation.
  • Export functionality to generate race xml's for the soda race application that illustrate the evolution of your models.
  • Parameterization of all features by means of a parameterization interface.

features - planned

  • Implementation of new phenotypes and selection strategies.
  • Distributed evaluation, by means of vgrid.

webstart

Here you can start the wodka using java webstart to breed your own soda robots.

Click here to start wodka.

When the application is running follow the instruction below.

  • Click the new button to create a new genetic algorithm.
  • Select a phenotype, and a selection strategy and/or any other parameter on the dialog. Then click OK. You may leave the default settings if you like.
  • Click start on the wodka breeder.
  • Wait for some generations to be finished. This could take several minutes. Waiting for hours will improve the results.
  • Save your results using the save button.
  • Export your results using the result button. The exported results may be imported into the soda racer to see your robots 'alife'.

checklist if wodka does not start

  • Make sure that java webstart is installed on your computer. Details see on the sun homepage
  • Perhaps you are behind a firewall and you access the internet using a proxy server. In that case you have to define the proxy server in the java webstart manager.

download

If you want to run wodka without java webstart the following prerequisites must be fulfilled:

  • Java 2 runtime must be installed on your computer. If you have no java runtime you get it free from the sun website. Calling the java virtual machine should be in your system search path. That means it should be possible to call 'java' on any working directory on your computer. This is usually the case if you install java on windows using the installer. On linux you may have to adopt the $PATH system variable.
  • It must be possible to start the soda race application on your computer. This requires basically that java webstart is installed on your computer. Details see on the sun homepage
    • .

Just execute the wodka_setup.exe and you will be able to run wodka on your local machine.

Download the the latest wodka release from the wodka download page and follow the instructions below.

  • Extract the zip file to any location on your computer. E.g. c:\program files\ on windows or /user/local on linux. Then start one of the batch files included. Breeder.bat or any .bat file on windows breeder.sh or any .sh file on linux (Attention I do not have a linux installation at the moment -> the .sh files are not tested and their execution might fail. Anyway it should be no problem to adapt them. They are quit simple). Eventually you will have to adopt the batch files for your computer configuration but it should not be necessary.
  • Click start ... . Follow the instructions for webstart.
